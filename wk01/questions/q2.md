What would be the worst case time-complexity for naively computing the Z-values
for a string str[1 .. n]?

Instead of using the powerful techniques (case 2a, 2b, & 2c) of the Z algorithm
to save time that would make it O(n + m) = O(n). 
    We would have to do naive comparisons for each substring for all i in the
string. This means that it would be polynomial time at O(n * m) = O(n^2) in the
worst case.
